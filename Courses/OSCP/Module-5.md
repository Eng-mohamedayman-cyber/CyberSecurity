# Introduction to Windows

Windows is a dominant operating system within corporate environments. Understanding its architecture, administrative command-line interfaces, and privilege models is a fundamental requirement for active directory exploitation and local post-compromise activities.

## Key Windows Architecture Concepts

* **User Mode vs. Kernel Mode:** User Mode isolates user-facing applications so they cannot corrupt core system files. Kernel Mode grants unprivileged, absolute access to core hardware resources and operating system instructions.
* **Windows API (Win32 API):** The primary programming interface used by software to interact directly with the Windows Kernel and execute subsystem tasks.
* **Security Identifiers (SIDs):** Unique alpha-numeric tracking strings utilized by the OS to identify users, groups, and computer accounts for credential auditing.
* **Access Tokens:** Digital keys generated by the system upon user authentication. They contain the user's SID, group SIDs, and operational privileges, and dictate access to objects.

---

## Command Line Interfaces (CLI): CMD vs. PowerShell

Windows provides two core environments for systems automation and administrative execution:
1. **Command Prompt (CMD):** The classic, legacy environment primarily managing simple batch execution and standard configurations.
2. **PowerShell:** A powerful task-automation framework built on top of the .NET ecosystem. It processes structured objects rather than plain text data streams, making it a critical asset for penetration testers.

---

## Basic File & Directory Navigation (CMD vs. PowerShell)

| Task | Command Prompt (CMD) | PowerShell (Cmdlet) |
| ---- | -------------------- | ------------------- |
| **Print Current Path** | `echo %cd%` | `Get-Location` (or `pwd`) |
| **List Directory Contents** | `dir` | `Get-ChildItem` (or `ls`) |
| **Change Directory** | `cd` | `Set-Location` (or `cd`) |
| **Create a Folder** | `mkdir <Folder_Name>` | `New-Item -ItemType Directory <Name>` |
| **Read File Contents** | `type <File_Name>` | `Get-Content <File_Name>` (or `cat`) |
| **Copy Files/Folders** | `copy <Source> <Dest>` | `Copy-Item <Source> <Dest>` |
| **Move/Rename Files** | `move <Source> <Dest>` | `Move-Item <Source> <Dest>` |
| **Delete Files** | `del <File_Name>` | `Remove-Item <File_Name>` (or `rm`) |

## System Enumeration & Information Gathering

These utilities are essential for localized post-exploitation scanning to identify the underlying operating system build and security posture:

* `systeminfo` : Generates a comprehensive summary of OS properties, system architecture, and applied hotfixes/patches.
* `whoami` : Returns the identity and privileges of the current session user.
* `whoami /priv` : Lists the administrative execution privileges granted to the current access token.
* `whoami /groups` : Displays the security groups that the current user belongs to.
* `hostname` : Displays the specific network name assigned to the system.

---

## Networking & Connection Auditing

* `ipconfig /all` : Generates detailed mapping for all network adapter cards, physical MAC addresses, and DNS configurations.
* `netstat -ano` : Audits active TCP/UDP connections, listening network ports, and links them to specific process IDs (PIDs).
* `route print` : Outputs the localized IP routing tables maintained by the target operating system.
* `net view` : Performs active network discovery to check for shared systems or accessible target systems within the domain environment.

---

## User & Group Account Administration

Managing system objects requires an elevated administrative command shell:

### Account & Group Scanning:
* `net user` : Compiles a quick list of all local accounts configured on the operating system.
* `net user <Username>` : Generates granular details about a specific account, showing group membership and password change policies.
* `net localgroup` : Queries all security groups available on the host system.
* `net localgroup Administrators` : Lists all local accounts holding full host management privileges.

### Modifying Accounts:
* `net user <Username> <Password> /add` : Creates a new local user account on the target machine.
* `net localgroup Administrators <Username> /add` : Appends a local account into the local Administrators group.
* `net localgroup "Remote Desktop Users" <Username> /add` : Grants RDP connection privileges to a specific user.

---

## File System Security (NTFS Permissions)

The **NTFS** file system tracks object security utilizing **Access Control Lists (ACLs)** and individual Access Control Entries (ACEs).

* `icacls <Path>` : Displays the exact permissions and security context configurations set on a specific file or directory.
* `icacls <Path> /grant <Username>:F` : Alters file permissions to grant **Full Control (F)** access to a targeted user.
* `icacls <Path> /deny <Username>:R` : Sets an explicit **Read (R)** denial rule against a specified account.

---

## Local Service & Process Management

* `tasklist` : Displays an active inventory of running background programs and services.
* `taskkill /PID <ID> /F` : Forcefully terminates a running system process by specifying its unique Process ID.
* `Get-Service` *(PowerShell)* : Lists all local system services alongside their operational states (Running, Stopped).
* `Start-Service -Name <Service>` *(PowerShell)* : Instructs a specified host service to initialize immediately.
* `Stop-Service -Name <Service>` *(PowerShell)* : Safely drops a running host service session.

---

## Windows Management Instrumentation (WMI)

WMI allows administrative queries to retrieve configuration metrics from deep within the OS infrastructure:

* `wmic qfe get Caption,Description,HotFixID` : Collects an overview of all installed patches and hotfixes (useful to look for exploit opportunities).
* `wmic product get name,version` : Identifies all installed software products currently managed on the system.
* `wmic service where "state='running'" get name, displayname` : Returns a list of all actively running backend services via a WMI filter query.
